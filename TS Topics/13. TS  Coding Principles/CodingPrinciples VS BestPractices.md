# Difference Between Coding Principles and Coding Best Practices

| Coding Principles                            | Coding Best Practices                                    |
| -------------------------------------------- | -------------------------------------------------------- |
| High-level design guidelines                 | Day-to-day coding habits                                 |
| Focus on software design and maintainability | Focus on writing clean, safe, and readable code          |
| Language-independent                         | Often language-specific                                  |
| Help decide _how to design_ code             | Help decide _how to write_ code                          |
| Examples: SOLID, DRY, KISS                   | Examples: Avoid `any`, use `const`, enable `strict` mode |

### Simple Analogy

Imagine you're building a house:

- **Coding Principles** = The architectural rules (strong foundation, proper room layout, good ventilation).
- **Best Practices** = The construction habits (use quality materials, measure accurately, keep the site clean).

You need both to build a good house.
