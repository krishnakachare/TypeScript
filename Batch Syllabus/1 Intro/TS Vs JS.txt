JavaScript is traditionally called an interpreted language because its code is executed by a JavaScript engine at runtime rather than being compiled into a standalone executable beforehand. However, modern JavaScript engines like V8 use Just-In-Time (JIT) compilation, converting frequently executed code into machine code for better performance. Therefore, JavaScript is more accurately described as an interpreted language with JIT compilation.
